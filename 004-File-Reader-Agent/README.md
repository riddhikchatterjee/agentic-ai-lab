Purpose:
Build an AI-powered document analysis agent that can read text-based files, understand their contents, and perform different user-selected actions such as summarization, analysis, key point extraction, beginner-friendly explanations, and custom question answering.

Concepts learned:

Python Concepts
- User Input (input)
- File Handling (open, read)
- Context Managers (with)
- Exception Handling (try/except)
- Conditional Logic (if/elif/else)
- String Manipulation
- Variables and Data Flow

AI Engineering Concepts
- Document Processing
- Context Injection
- Prompt Construction
- Task + Context Design Pattern
- User Intent Handling
- AI-Powered Document Analysis
- Dynamic Prompt Generation

Software Engineering Concepts
- Error Handling
- Input Validation
- User Experience Design
- Modular Thinking
- Interactive Workflows
- File Path Management

Key Takeaways:
- AI systems become more useful when they can process external data instead of relying only on user messages.
- The quality of AI responses depends heavily on the combination of user intent and contextual information.
- Professional applications should handle errors gracefully instead of crashing.
- The same AI model can perform different tasks by dynamically changing prompts.
